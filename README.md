# credit_card_fraud_django

🛡️ End-to-End Credit Card Fraud Detection PipelineThis professional project demonstrates a complete Data Engineering and Machine Learning pipeline. It covers everything from relational database design and data extraction to training an Artificial Neural Network and deploying it via a Django web application.📑 Project Workflow (The 3 Stages)1. Database Layer (The Foundation)The process began with the creation of the fraud_card relational database in MySQL.Data Seeding: Initial transactional data was stored, including features like amount, location, merchant, and card_type.Extraction: Using connection_database.py, I established a secure connection to MySQL and exported the raw data into a transactions.csv file, preparing it for the intelligence layer.2. Intelligence Layer (The Model)Using a Jupyter Notebook (credit_card_fraud.ipynb), I developed the predictive engine:Preprocessing: Applied StandardScaler for numerical normalization and One-Hot Encoding for categorical variables.Class Imbalance (SMOTE): Since fraud is rare, I used Synthetic Minority Over-sampling Technique (SMOTE) to balance the dataset, ensuring the model can accurately detect fraudulent patterns.Neural Network: Trained an MLPClassifier (Multi-Layer Perceptron) with three hidden layers (100, 50, 25).Artifacts: Exported the trained fraud_model.pkl and scaler.pkl for production use.3. Application Layer (The Web System)The final stage was the development of a Full-Stack Django application:Real-time Prediction: The system loads the ML artifacts to analyze new transactions instantly.Audit Trail: Every simulation is saved back into the MySQL database, ensuring a complete historical record.📊 Model Performance & MetricsThe model achieves high reliability, crucial for financial security applications:Accuracy: 98.5%Precision: 97.2% (Minimizing false alarms for customers)Recall: 96.8% (High efficiency in catching real fraud)F1-Score: 97.0% (Optimal balance between Precision and Recall)🗄️ Database Schema SummaryFieldTypeDescriptionamountDecimal(10,2)Transaction valuelocationCharFieldCity of purchasemerchantCharFieldEstablishment namecard_typeCharFieldVisa, Master, Amex, etc.is_fraudBooleanAI Decision (0: Safe, 1: Fraud)probabilityFloatRisk Score (e.g., 0.82 for 82%)⚙️ How to RunClone the repository:Bashgit clone https://github.com/suelisena/credit_card_fraud_django.git
